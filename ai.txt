## PERMITTED USES

- Humans may read, reference, and cite this content for research,
  evaluation, or learning purposes, with attribution.

- Search engines may index this repository and associated website
  content for the purpose of discovery and linking, subject to any
  platform-level restrictions in robots.txt.

- User-initiated AI retrieval is permitted where an end user
  directs an AI system to retrieve, explain, summarize, or answer
  questions about the framework for their own study, evaluation,
  or reference — provided this retrieval is initiated on demand
  by a human user and does not involve ingesting the framework
  content into a persistent model, index, or product system.


## PROHIBITED USES

The following are explicitly NOT permitted without prior written
authorization:

- Training, fine-tuning, aligning, or otherwise developing any
  AI or machine learning model using any part of this repository —
  including documentation, prompt logic, architectural patterns,
  specifications, naming conventions, or source files.

- Including this content in datasets, corpora, or data pipelines
  intended for AI model development, evaluation, or benchmarking,
  whether commercial or otherwise.

- Automated scraping of this content for the purpose of model
  pre-training, instruction tuning, RLHF data collection, or
  embedding generation.

- Ingesting this content into retrieval-augmented generation (RAG)
  systems operating as part of a product or platform, including
  embedding databases, vector stores, and system prompt stores.

- Integrating this content into an AI assistant's system prompt,
  agent instructions, or persistent model behavior — whether in a
  commercial, organizational, or developer-tool context.

- Replicating, adapting, or reverse-engineering the framework's
  prompt logic, agent architecture, or design system into competing
  products or systems.

- Redistributing this content in modified form to circumvent
  these restrictions.


## DISTINCTION: INDEXING vs. TRAINING vs. USER-INITIATED RETRIEVAL

Search engine crawlers that index content for the purpose of
returning it in search results (e.g., Googlebot) are permitted.

AI crawlers that collect content for model training, dataset
construction, or systematic embedding are not permitted,
regardless of the organization operating them.

User-initiated retrieval — where a person explicitly asks an
AI system about this framework for learning or evaluation — is
permitted. Automated, background, or product-integrated ingestion
is not permitted, even if framed as serving user needs.


## TECHNICAL ENFORCEMENT (robots.txt)

This policy is technically reflected in /robots.txt:

- Search-indexing crawlers (e.g. Googlebot, Bingbot, Applebot,
  DuckDuckBot) and on-demand/user-initiated AI retrieval crawlers
  (e.g. ChatGPT-User, OAI-SearchBot, Claude-User, Claude-SearchBot,
  PerplexityBot, Perplexity-User) are allowed.

- AI training and dataset-construction crawlers (e.g. GPTBot,
  ClaudeBot, anthropic-ai, Google-Extended, Applebot-Extended,
  Meta-ExternalAgent, Amazonbot, Bytespider, CCBot, and other known
  data-harvesting bots) are disallowed.

robots.txt is a voluntary technical signal honored by compliant
crawlers. It does not limit, and is not a substitute for, the rights
reserved in this file and in LICENSE.md. A crawler ignoring
robots.txt does not gain any authorization that this file and
LICENSE.md do not grant.

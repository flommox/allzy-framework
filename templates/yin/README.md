# Yin Templates

This folder contains reusable Yin structures for human intent, product meaning, workflows, roles, and scope boundaries.

## Use This Folder First

Choose the scope first:

- Use [`yin-page/02_yin_template.md`](yin-page/02_yin_template.md) for a Full Yin Page.
- Use [`compact-yin-page/02_yin_template_compact.md`](compact-yin-page/02_yin_template_compact.md) for a Compact Yin Page.

The numbered template file in each leaf folder is the canonical structure.

## Nearby Differences

- Full Yin is for richer, longer-lived intent documentation.
- Compact Yin is for smaller bounded work that still needs human context.
- Yin does not define deterministic technical execution rules.

Use [`../yang/yang-core-module/03_yang_template.md`](../yang/yang-core-module/03_yang_template.md) when the artifact must be a Full Yang contract.

Use [`../yin-yang/yin-yang-document/04_yin_yang_template.md`](../yin-yang/yin-yang-document/04_yin_yang_template.md) or [`../yin-yang/yin-yang-compact/04_yin_yang_template_compact.md`](../yin-yang/yin-yang-compact/04_yin_yang_template_compact.md) when intent and execution rules must live together.

There is no standalone Yang Compact.
